Switch Statements

Switch

The switch statement is used to execute one block of code among many options based on the value of an expression. It is more readable than a series of if-else statements when dealing with multiple conditions.

There are two types of switch statements

  1. Normal Switch
  2. Advance Switch

  1. Java Switch Statement

    1. Definition

      The switch statement in Java is a control flow statement that allows you to choose among multiple options based on the value of a variable or expression. It provides an efficient way to handle multiple conditions by using a single variable to select between several options.

    2. Detailed Working
      1. Syntax 
        switch (expression) {
    case value1:
        // code to be executed if expression equals value1
        break;
    case value2:
        // code to be executed if expression equals value2
        break;
    case value3:
        // code to be executed if expression equals value3
        break;
    default:
        // code to be executed if expression doesn't match any case
}
      2. Component
        • Expression The variable or expression whose value is tested against the cases.
        • Case A block of code that executes if the expression matches the case value. Each case is checked sequentially.
        • Break A statement that terminates the switch block, preventing fall-through to subsequent cases.
        • Default A block of code that executes if none of the case values match the expression. This block is optional.
      3. Flow
        • Evaluate the expression.
        • Compare the value of the expression with each case value sequentially.
        • If a match is found, execute the corresponding block of code.
        • Use the `break` statement to exit the switch block. If `break` is omitted, execution continues with the next case (fall-through).
        • If no cases match and a `default` block is provided, execute the `default` block.
      4. Example of Switch Cases
        • switch (day) { case 1: ... }
        • switch (color) { case "red": ... }
        • switch (grade) { case 'A': ... }
    3. E-R Diagram

      An ER diagram is not applicable to the switch statement since it pertains to control flow rather than entity relationships.

    4. Program
      public class SwitchExample {
public static void main(String[] args) {
    int day = 3;

    // Using a switch statement to determine the day of the week
    switch (day) {
        case 1:
            System.out.println("Monday");
            break;
        case 2:
            System.out.println("Tuesday");
            break;
        case 3:
            System.out.println("Wednesday");
            break;
        case 4:
            System.out.println("Thursday");
            break;
        case 5:
            System.out.println("Friday");
            break;
        case 6:
            System.out.println("Saturday");
            break;
        case 7:
            System.out.println("Sunday");
            break;
        default:
            System.out.println("Invalid day");
            break;
    }

    System.out.println("This line always executes.");
}
    5. Real-life Example
      • Condition Based on the day of the week (e.g., day = 3).
      • Action If the day is 1, you say "Monday". If the day is 2, you say "Tuesday". If the day is 3, you say "Wednesday", and so on. If the day is not within the range of 1 to 7, you say "Invalid day".

      Example 1 Nested Switch for Days and Months

      public class NestedSwitchExample1 {
        public static void main(String[] args) {
            int day = 2;
            int month = 4;
     
            switch (month) {
                case 4:
                case 6:
                case 9:
                case 11:
                    switch (day) {
                        case 30:
                            System.out.println("The day is the last day of the month.");
                            break;
                        default:
                            System.out.println("The day is not the last day of the month.");
                    }
                    break;
                    default:
                        System.out.println("The month is not one with 30 days.");
        }
    }
}
      Output
      The day is not the last day of the month.

  2. Advanced Java Switch Statement

    1. Definition

      The advanced switch statement in Java introduces enhanced features starting from Java 12 as a preview feature and became a standard feature in Java 14. This includes switch expressions, which allow the switch statement to return values, and the ability to use multi-line code blocks. These enhancements provide greater flexibility and functionality in control flow handling.

    2. Detailed Working
      1. Switch Expressions 
        int day = 3;
String dayName = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    case 4 -> "Thursday";
    case 5 -> "Friday";
    case 6 -> "Saturday";
    case 7 -> "Sunday";
    default -> "Invalid day";
};
System.out.println(dayName);
      2. Block Body with Multiple Statements 
        int day = 3;
switch (day) {
    case 1: 
    case 2: 
        System.out.println("Weekday");
        break;
    case 3: 
    case 4: 
    case 5:
        System.out.println("Midweek");
        break;
    case 6: 
    case 7: 
        System.out.println("Weekend");
        break;
    default:
        System.out.println("Invalid day");
        break;
}
      3. Enhanced Switch Statements 
        int day = 3;
switch (day) {
    case 1, 2 -> System.out.println("Weekday");
    case 3, 4, 5 -> System.out.println("Midweek");
    case 6, 7 -> System.out.println("Weekend");
    default -> System.out.println("Invalid day");
}
      4. Label and Yield Statement 
        String dayType = switch (day) {
    case 1, 2 -> {
        System.out.println("Weekday");
        yield "Weekday";
    }
    case 3, 4, 5 -> {
        System.out.println("Midweek");
        yield "Midweek";
    }
    case 6, 7 -> {
        System.out.println("Weekend");
        yield "Weekend";
    }
    default -> {
        System.out.println("Invalid day");
        yield "Invalid";
    }
};
System.out.println(dayType);
      5. Flow
        • Evaluate the expression in the `switch` statement.
        • Compare the value of the expression with each case label.
        • Execute the corresponding block of code if a match is found.
        • Use the `break` statement to exit the `switch` block or let it fall through if `break` is omitted.
        • With switch expressions, return a value directly from the `switch` block.
        • With enhanced switch statements, use `->` and block bodies for multiple statements and the `yield` keyword to return values from cases.
    3. E-R Diagram

      An ER diagram is not applicable to the switch statement since it pertains to control flow rather than entity relationships.

    4. Program
      public class AdvancedSwitchExample {
  public static void main(String[] args) {
      int day = 3;

      // Using a switch expression with the new features
      String dayName = switch (day) {
          case 1 -> "Monday";
          case 2 -> "Tuesday";
          case 3 -> "Wednesday";
          case 4 -> "Thursday";
          case 5 -> "Friday";
          case 6 -> "Saturday";
          case 7 -> "Sunday";
          default -> "Invalid day";
      };
      System.out.println(dayName);

      // Using block bodies with multiple statements
      switch (day) {
          case 1: 
          case 2: 
              System.out.println("Weekday");
              break;
          case 3: 
          case 4: 
          case 5:
              System.out.println("Midweek");
              break;
          case 6: 
          case 7: 
              System.out.println("Weekend");
              break;
          default:
              System.out.println("Invalid day");
              break;
      }

      // Using enhanced switch statement with labels and yield
      String dayType = switch (day) {
          case 1, 2 -> {
              System.out.println("Weekday");
              yield "Weekday";
          }
          case 3, 4, 5 -> {
              System.out.println("Midweek");
              yield "Midweek";
          }
          case 6, 7 -> {
              System.out.println("Weekend");
              yield "Weekend";
          }
          default -> {
              System.out.println("Invalid day");
              yield "Invalid";
          }
      };
      System.out.println(dayType);
  }
      Output
      Wednesday
Midweek
Midweek
Midweek

      Example 2 Enhanced Switch with Multiple Cases

      public class EnhancedSwitchExample2 {
        public static void main(String[] args) {

      Example 3 Enhanced Switch with Block

      public class EnhancedSwitchExample3 {
        public static void main(String[] args) {
            String month = "February";
            int days = switch (month) {
                case "January", "March", "May", "July", "August", "October", "December" -> 31;
                case "April", "June", "September", "November" -> 30;
                case "February" -> {
                    System.out.println("Checking if leap year...");
                    boolean leapYear = false; // Simplified leap year check
                    yield leapYear ? 29 : 28;
                }
                default -> throw new IllegalArgumentException("Unknown month: " + month);
            };
            System.out.println("Days in " + month + ": " + days);
        }
    }
      Output
      Checking if leap year...
            Days in February: 28

      Example 5 Nested Enhanced Switch

      public class EnhancedSwitchExample5 {
        public static void main(String[] args) {
            int year = 2023;
            int month = 2;
            int day = 28;
     
            String dayType = switch (month) {
                case 1, 3, 5, 7, 8, 10, 12 -> "31-day month";
                case 4, 6, 9, 11 -> "30-day month";
                case 2 -> switch (day) {
                    case 29 -> "Leap year February";
                    default -> {
                        boolean leapYear = (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
                        yield leapYear ? "Leap year February" : "Non-leap year February";
                    }
                };
                default -> "Invalid month";
            };
            System.out.println("The month and day type: " + dayType);
        }
    }
      Output
      The month and day type: Non-leap year February

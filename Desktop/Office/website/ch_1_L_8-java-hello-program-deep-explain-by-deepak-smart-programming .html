Working Of main method

  1. The Java main Function

    The Java main function is a critical component for executing Java programs. It serves as the entry point and is fundamental for understanding how Java applications are started. This comprehensive guide will explore each keyword used in the main function, including the class keyword which is crucial for context.

    1. The Java main Method: An Overview

      The main method is the entry point for any standalone Java application. It serves as the starting point where the Java Virtual Machine (JVM) begins the execution of a program. To appreciate its importance, let’s dissect its structure and each of its components.

      1. Structure of the main method with java class.

        Here is the typical structure of the main method: 

        public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

        In this example, we have a class named HelloWorld, and within this class, there is a main method. The method prints "Hello, World!" to the console.

      2. Breaking Down the main Method
        1. 1. public Keyword
          • The public keyword is an access modifier that makes the main method accessible from any other class. In Java, access modifiers determine the visibility of classes, methods, and other members. By declaring the main method as public, we ensure that the JVM can invoke it from outside the class.
          • The main method, which serves as the entry point of any Java application, must be public. This is necessary because the Java Virtual Machine (JVM) needs to access this method from anywhere in the system.
        2. 2. class keyword
          • The class keyword is fundamental in Java as it defines a blueprint for creating objects. It encapsulates data and methods, including the main method, providing structure and organization.
          • A Java class encapsulates data and methods. The main method is typically enclosed within a class. Here’s a breakdown of the structure:
        3. 3. HelloWorld
          • HelloWorld is an identifier in this context. Identifiers are names used to identify variables, methods, classes, and other elements in Java.
            • The class name should follow Java's naming conventions and rules for identifiers:
            • Must start with a letter, underscore (_), or dollar sign ($).
            • Subsequent characters can be letters, digits, underscores, or dollar signs.
            • Should not be a reserved keyword.
        4. 4. { } (Curly Braces)
          • Curly braces { } define a block of code. In Java, they are used to group statements and to define the boundaries of classes, methods, loops, and conditionals.
          • Every opening brace { must have a corresponding closing brace }. This pairing is crucial for maintaining code structure and logic.
        5. 5. static Keyword

          The static keyword denotes that the main method is a class method, not an instance method. This means it can be called directly by the JVM without creating an instance of the class.

        6. 6. void Keyword

          The void keyword specifies that the main method does not return any value. It performs operations, such as executing code or printing output, without producing a return result.

        7. 7. main() Method Name

          The name main is a special identifier recognized by the JVM as the entry point of the program. It must be exactly main to conform to Java standards.

        8. 8. String[] args Parameter

          The String[] args parameter is an array of String objects used to pass command-line arguments to the program. This allows users to provide input data when the program starts.

        9. 9. System

          System is a predefined class in the java.lang package. It provides various utility methods and fields that are globally accessible.

        10. 10. out
          • out is a static variable of the PrintStream class within the System class. It represents the standard output stream, typically the console.
          • There are two more "static variable" of "PrintStream" class present in "System" class in (for input), err (for error)
        11. 11. println
          • println() is a method of the PrintStream class used to print a line of text to the console. It appends a newline character after the text, moving the cursor to the next line.
          • There are two more method related to printing print(), printf()
      3. What changes are acceptable for main method
        • The order of modifiers can be changed, so instead of “public static” we can write “static public”.

          • We can declare “String[] args” in any acceptable form, which are given below :

          • String[] args
          • String []args
          • String args[]
        • Instead of “args” we can write any valid java identifier.
        • We can replace “String[] args” with varargs (Variable Arguments) parameter i.e. we can write “String... args”.
        • We can use “final”, “synchronized” and “strictfp” modifiers with main method.

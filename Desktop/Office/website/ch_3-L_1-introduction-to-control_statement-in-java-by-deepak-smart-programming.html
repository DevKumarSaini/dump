Introduction to Control Statements

Introduction to Control Statements in Java

Control statements in Java allow you to dictate the flow of execution in your programs. They are essential for implementing logic and handling various conditions and loops in your code. Java provides several types of control statements, including conditional statements, looping statements, and branching statements. Here’s an overview of each type:

Types of conditional statement in java.

  1. Decision Making Statements / Conditional Statements / Selection Statements

    • if-statements

      • if
      • if-else
      • else-if
    • switch statement and enhanced switch
    • ternary statement

  2. Looping statements / Iterations / Control flow

    • for
    • while
    • do-while
    • for-each

  3. Jump statement / Branching statements

    • break
    • continue
    • return

  4. Control flow Enhancements

    • Labels with (Break and continue)
    • goto
    • assert

Brief definition

  • Decision making

    Decision-making in Java refers to the process of choosing a particular path of execution based on the evaluation of conditions. It involves using control statements that allow the program to make decisions and execute different code blocks based on these conditions.

  • Control Flow

    Control flow in Java refers to the order in which individual statements, instructions, or function calls are executed or evaluated. The control flow of a program is dictated by control statements that determine the path the execution will take based on conditions or loops.

  • Jump Statements

    Jump statements are used to control the flow of execution by breaking out of loops or switching to different parts of the program. They alter the normal sequence of execution and are typically used to exit loops or skip certain parts of code.

  • Control flow enhancements

    Control flow enhancements are advanced features that provide additional control over the execution flow, allowing for more complex or refined program logic. This includes labeled statements, the goto keyword (though not used in Java), and assertions.

    1. Labele

      Allows you to assign labels to code blocks and use these labels with break and continue statements to control flow more precisely in nested loops

      outerLoop: for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 5; j++) {
        if (i == 2 && j == 3) {
            break outerLoop;  // Breaks out of the outer loop
        }
        System.out.println(i + " " + j);
    }
}
    2. goto

      Reserved for future use in Java and does not have any functionality in the language. It was included in the syntax for historical reasons but is not used in practice.

    3. assert

      Used for debugging purposes to test assumptions about the program’s state. If an assertion fails (i.e., evaluates to false), an AssertionError is thrown.

Block in java

In Java, a block is a group of statements enclosed within curly braces {}. Blocks are used to group statements and define the scope of variables. They help manage the execution flow and variable visibility in a program. There are several types of blocks in Java, each serving different purposes:

Types of block
  1. Method Block

    A method block is a block of code that defines the body of a method. It is executed when the method is called.

    public void displayMessage() {
    // Method block starts here
    System.out.println("Hello, World!");
    // Method block ends here
}
  2. Instance Block (Non-static Block)

    An instance block is a block of code that is executed every time an instance of the class is created. It is used to initialize instance variables.

    public class MyClass {
    {
        // Instance block starts here
        System.out.println("Instance block executed.");
        // Instance block ends here
    }
    
    public MyClass() {
        System.out.println("Constructor executed.");
    }
    
    public static void main(String[] args) {
        MyClass obj = new MyClass();
    }
}
  3. Static Block

    A static block is a block of code that is executed only once when the class is first loaded into memory. It is used to initialize static variables or perform class-level setup tasks.

    public class MyClass {
    static {
        // Static block starts here
        System.out.println("Static block executed.");
        // Static block ends here
    }
    
    public static void main(String[] args) {
        System.out.println("Main method executed.");
    }
}
  4. Local Block

    A local block is a block of code within a method or a constructor. It is used to create a new scope for variables, allowing you to declare variables that are limited to that block.

    public void method() {
    int a = 10;
    
    {
        // Local block starts here
        int b = 20;
        System.out.println("Inside local block: a = " + a + ", b = " + b);
    }
    
    // System.out.println(b); // Error: b cannot be resolved to a variable
}
  5. Synchronized Block

    A synchronized block is used to control access to a block of code by multiple threads to prevent concurrency issues. It ensures that only one thread can execute the block at a time.

    public class MyClass {
    private int count = 0;
    
    public void increment() {
        synchronized(this) {
            // Synchronized block starts here
            count++;
            System.out.println("Count: " + count);
            // Synchronized block ends here
        }
    }
}

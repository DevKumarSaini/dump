Jumping Statements

Break Statement

  • Definition

    The break statement in Java is used to exit from a loop or switch statement prematurely. When a break statement is encountered, the loop or switch is immediately terminated, and the program control continues with the statement following the loop or switch.

  • Detailed Working
    1. Syntax 
      break;
    2. Component
      • Loop or Switch BlockThe context in which the break statement is used. It can be used inside for, while, or do-while loops, and switch statements.
    3. Flow
      • When a break statement is executed inside a loop or switch, control is immediately transferred to the statement following the loop or switch block.
      • In loops, this means exiting the loop regardless of the loop condition.
      • In switch statements, this means exiting the switch block and continuing with the code that follows the switch statement.
    4. Example of Different Condition
      • When you want to terminate a loop based on a condition that is checked during each iteration, you can use break to exit the loop early.
      • When dealing with nested loops, you can use break to exit from the innermost loop and continue execution after the outer loop.
      • In scenarios where you need to terminate a loop if an invalid condition arises, break can be used to stop further processing.
      • When an infinite loop is used (e.g., while (true)), break allows you to terminate the loop based on a specific condition.
      • When dealing with processes that can encounter errors, break can be used to exit the loop to prevent further actions.

    public class BreakInLoopExample {

    Program 2 This program demonstrate the break statement using switch.

    public class BreakInSwitchExample {
    public static void main(String[] args) {
        int day = 3;
        
        switch (day) {
            case 1:
                System.out.println("Monday");
                break; // Exit the switch statement
            case 2:
                System.out.println("Tuesday");
                break; // Exit the switch statement
            case 3:
                System.out.println("Wednesday");
                break; // Exit the switch statement
            default:
                System.out.println("Invalid day");
        }
        
        System.out.println("Switch case completed.");
    }
}
    Output
    Wednesday
Switch case completed.
  • Real-life Example
    • In Loops Consider a search operation where you want to stop searching as soon as you find the target item. Once found, you use break to exit the loop and avoid unnecessary further searching.
    • In Switch Statements When evaluating multiple cases, break ensures that only the code for the matched case is executed, and control exits the switch block, preventing fall-through to subsequent cases.
  • Use case
    • Terminate a Loop Early: Exit a loop as soon as a specific condition is met, such as finding a target value or meeting a stopping criterion.
    • Exit Nested Loops: Break out of multiple nested loops when a condition is satisfied, often used with labeled breaks to exit all levels of nested loops.
    • Stop an Infinite Loop: End an infinite loop (e.g., while (true)) based on a specific condition or user input.
    • Abort a Search Operation: Stop searching through a collection or dataset once the desired item is found.
    • Handle Error Conditions: Exit a loop when an error or exceptional condition occurs to prevent further processing or iterations.

Continue Statement

  • Definition

    The continue statement in Java is used to skip the current iteration of a loop and proceed with the next iteration. When a continue statement is encountered, the remaining statements inside the loop for the current iteration are skipped, and the loop proceeds to the next iteration based on the loopâ€™s condition.

  • Detailed Working
    1. Syntax 
      continue;
    2. Component
      • Loop BlockThe context in which the continue statement is used. It can be used inside for, while, or do-while loops.
    3. Flow
      • When a continue statement is executed inside a loop, the remaining statements of the loop body are skipped for the current iteration.
      • The loop then evaluates its condition (in while and do-while loops) or proceeds to the next iteration (in for loops).
      • If the condition is still true, the loop continues with the next iteration; otherwise, it terminates.
    4. Example of Different Condition
      • When you want to skip certain iterations of a loop based on a condition (e.g., skipping even numbers in a list).
      • When processing a list of items where some items do not need to be processed, you can use continue to skip those items.
      • In scenarios where you need to skip processing when a certain condition is met (e.g., skipping over null values in an array).
      • When filtering out unwanted values in a loop and only processing those that meet specific criteria.
  • E-R Diagram
    Diagram required.
  • Program

    Program 1 This program demonstrate the continue statement in for loop.

    public ContinueInForLoopExample {
    public static void main(String[] args) {
        // Using a continue statement to skip even numbers
        for (int i = 1; i <= 10; i++) {
            if (i % 2 == 0) {
                continue; // Skip the rest of the loop body if i is even
            }
            System.out.println("Odd number: " + i);
        }
    }
}
    Output
    Odd number: 1
Odd number: 3
Odd number: 5
Odd number: 7

    Program 2 This program demonstrate the continue statement in while loop.

    public class ContinueInWhileLoopExample {
    public static void main(String[] args) {
        int number = 1;

        // Loop through numbers from 1 to 10
        while (number <= 10) {
            if (number % 3 == 0) {
                number++;
                continue; // Skip the rest of the loop body if number is divisible by 3
            }
            System.out.println("Number: " + number);
            number++;
        }
    }
}
    Output
    Number: 1
Number: 2
Number: 4
Number: 5
Number: 7
Number: 8
Number: 10
  • Real-life Example
    • In LoopsConsider a scenario where you need to process a list of records, but some records should be skipped based on specific criteria (e.g., skipping records with missing data). You use continue to skip the processing of those records and continue with the next one.
    • In Filtering OperationsWhen filtering data, you might want to skip certain items that do not meet the criteria. continue helps in bypassing the non-relevant items and focusing on those that meet the requirements.
  • Use case
    • Skip Iterations Based on ConditionUse continue to skip certain iterations of a loop where specific conditions are met, such as skipping every second item in a list.
    • Filter DataIn scenarios where only certain data needs processing, continue helps in filtering out unwanted data by skipping those iterations.
    • Handle Special CasesWhen dealing with special cases or exceptions in loop processing, continue allows you to bypass problematic cases and continue with the rest of the data.
    • Avoid Redundant ProcessingUse continue to avoid redundant or unnecessary processing by skipping over iterations that do not need to be executed.

Return Statement

  • Definition

    The return statement in Java is used to exit a method and optionally return a value to the caller. It terminates the execution of the method in which it is used and passes control back to the point where the method was called. If the method has a return type other than void, the return statement must return a value of that type.

  • Detailed Working
    1. Syntax 
      return; // For void methods
return value; // For methods with a return type other than void
    2. Component
      • Void Method A method that does not return any value. In this case, return is used to exit the method without returning a value.
      • Non-Void Method A method that returns a value. In this case, return is used to exit the method and return a value of the specified type.
    3. Flow
      • When a return statement is executed, the method immediately exits and returns to the caller.
      • If the method is of type void, the return statement does not return any value; it simply exits the method.
      • If the method has a return type (e.g., int, String), the return statement must provide a value of that type.
    4. Example of Different Condition
      • When you want to terminate a method early and optionally return a result, use return to exit the method and provide the result.
      • In conditional methods where you need to exit based on specific conditions (e.g., if a parameter is invalid), return can be used to stop further execution and return an appropriate value or error.
      • When implementing methods that perform calculations or data processing, use return to provide the final result back to the caller.
      • In recursive methods, return is used to pass the result of recursive calls back up the call stack.

  • Real-life Example
    • In MethodsWhen performing operations or calculations, such as computing a total or checking conditions, return allows you to send the result back to the caller or exit the method when a specific condition is met.
    • In Error Handling If a method encounters an error or invalid input, return can be used to exit the method early and return an error code or message.
  • Use case
    • Terminate Method Execution Use return to exit a method as soon as a result is computed or when certain conditions are met, avoiding unnecessary code execution
    • Return ResultsWhen a method performs a calculation or data retrieval, use return to pass the result back to the caller.
    • Early ExitIf a method is processing data and encounters an invalid state or error, return can be used to exit and handle the error or state appropriately.
    • Recursive MethodsIn recursive methods, return is essential for passing the result of recursive calls back up the call stack.

    Program 1 
    public class ReturnInConditionalExample {
    public static void main(String[] args) {
        System.out.println("Factorial of 5 is: " + factorial(5));
        System.out.println("Factorial of -1 is: " + factorial(-1));
    }

    // Method to calculate factorial
    public static int factorial(int n) {
        if (n < 0) {
            return -1; // Return error code for invalid input
        }
        if (n == 0) {
            return 1; // Base case for recursion
        }
        return n * factorial(n - 1); // Recursive call
    }
}
    Output
    Factorial of 5 is: 120
Factorial of -1 is: -1

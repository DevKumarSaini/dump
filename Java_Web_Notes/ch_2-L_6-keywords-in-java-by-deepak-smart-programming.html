Keywords in java

Java Keywords

In Java, keywords are reserved words that have a predefined meaning in the Java programming language. They are integral to the syntax and structure of the language, and their meanings are established by the Java language specification. Keywords cannot be used as identifiers (such as variable names, function names, or class names) because they are reserved by the language.

Category Keyword Description
Access Control
1 private Defines a member of a class that is accessible only within the class.
2 protected Defines a member that is accessible within the same package and by subclasses.
3 public Defines a member that is accessible from any other class.
Class and Object
4 abstract Declares a class or method that cannot be instantiated or must be implemented by subclasses.
5 class Defines a new class.
6 interface Defines an interface.
7 extends Indicates that a class or interface inherits from another class or interface.
8 implements Indicates that a class implements an interface.
9 final Defines constants, prevents method overriding, and prevents class inheritance.
10 static Defines a member that belongs to the class rather than instances.
11 this Refers to the current instance of a class.
12 super Refers to the superclass of the current object, used to access superclass methods and constructors.
13 record Defines a class that acts as a transparent carrier for immutable data.
14 sealed Restricts which classes can extend a class or implement an interface.
15 non-sealed Allows a sealed class to be extended.
16 transient Prevents serialization of a member variable.
Control Flow
17 break Exits from the current loop or switch statement prematurely.
18 continue Skips the current iteration of a loop and proceeds to the next iteration.
19 default Defines a default block of code in a switch statement to execute if no case matches.
20 do Defines a loop that executes a block of code once before checking the loop’s condition.
21 else Defines a block of code to execute if the if condition evaluates to false.
22 for Defines a loop that executes a block of code a specific number of times.
23 if Defines a conditional statement that executes a block of code if its condition evaluates to true.
24 switch Defines a multi-way branch statement based on the value of an expression.
25 while Defines a loop that executes as long as its condition evaluates to true.
26 yield Exits from a switch expression and returns a value.
Exception Handling
27 try Defines a block of code that is tested for exceptions.
28 catch Defines a block of code that handles exceptions thrown by a try block.
29 finally Defines a block of code that always executes after a try-catch block, regardless of exceptions.
30 throw Explicitly throws an exception.
31 throws Specifies the exceptions that a method might throw.
Data Types
32 byte Defines a variable of type byte (8-bit signed integer).
33 short Defines a variable of type short (16-bit signed integer).
34 int Defines a variable of type int (32-bit signed integer).
35 long Defines a variable of type long (64-bit signed integer).
36 float Defines a variable of type float (32-bit floating-point number).
37 double Defines a variable of type double (64-bit floating-point number).
38 char Defines a variable of type char (16-bit Unicode character).
39 boolean Defines a variable of type boolean (true or false).
Package Management
40 import Imports classes or entire packages into the current file.
41 package Defines a namespace for organizing classes and interfaces.
Modules and Packages
42 module Defines a module that groups related packages and resources.
43 requires Specifies a dependency on another module.
44 exports Specifies which packages are accessible from other modules.
45 opens Allows other modules to access the internal packages at runtime.
46 uses Specifies a service provider that is used by a module.
47 provides Specifies a service implementation provided by a module.
48 permits Defines permitted subclasses in a sealed class.
49 to Used in records to specify component names.
50 with Used in records to specify components in pattern matching.
Concurrency
51 synchronized Ensures that a block of code or method is accessed by only one thread at a time.
52 volatile Indicates that a variable’s value may be modified by different threads.
Other
53 assert Used for debugging purposes to test assumptions about the program’s state during runtime.
54 const Reserved for future use; not currently used in Java.
55 goto Reserved for future use; not currently used in Java.
56 native Specifies that a method is implemented in native code using JNI (Java Native Interface).
57 null Represents a null reference, indicating that a variable does not refer to any object.
58 return Exits from the current method and optionally returns a value.
59 void Specifies that a method does not return any value.
60 strictfp Ensures floating-point calculations adhere strictly to IEEE 754 standards.
61 transitive Allows transitive dependencies in module definitions.
62 var Local variable type inference for declaring variables.
63 when Used in pattern matching with records (Java 17+).
